What's New in Aiken Plugin

Version {{VERSION}}

This release turns Aiken support into a complete in-IDE workflow: project scaffolding, deep navigation, rich run integrations, artifact generation, parameterization flows, and polished daily UX from first file to final build output.

End-to-End Integration
Full loop inside JetBrains IDEs: edit, run, test, parametrize, and build artifacts.
Dedicated Runner Stack
Run checks, Build blueprint, Parametrize, Make artifacts, Clean artifacts.
Scaffold + Templates
New Project Wizard and New Aiken File templates ready for immediate coding.

Runner Integrations

Run checks

  • Output modes: tty, json, and ide integrated.
  • Structured tree by modules and tests with better labels and counters.
  • Warnings and errors are grouped as dedicated sections.
  • Detailed diagnostics are preserved with full context.
  • Selected test rerun and richer per-test details (including mem/cpu).
  • Improved match pattern UX and parser behavior for real CLI output streams.
[Screenshot placeholder: Run checks with tests + warnings/errors tree]

Build blueprint

  • Supports tty and IDE-integrated reporting.
  • Build-style result handling without running tests.
  • Clear success and failure messages with warning/error visibility.
[Screenshot placeholder: Build blueprint integrated output]

Parametrize

  • Both tty and IDE-integrated modes are supported.
  • Auto-repeat parameterization until no undefined parameters are left.
  • Smart completion handling for "no parameters left" terminal state.
  • Applied CBOR parameters are collected and shown for reuse.
  • Queue-based auto-applied CBOR input with comma/space parsing.
  • Rich GUI handling for nested data, constructors, lists, options, booleans, and opaque data input.
[Screenshot placeholder: Parametrize GUI with nested structures]

Make artifacts

  • Unified pipeline for script conversion, addresses, and policy output.
  • Per-artifact generation toggles (script, testnet address, mainnet address, policy).
  • Dynamic file templates with %module% and %validator%.
  • Template paths support subdirectories for advanced artifact layout.
  • Aggregates multiple module/validator targets in one run.
  • Result panel with copy-friendly fields and saved-file summary.
[Screenshot placeholder: Make artifacts result view + saved files]

Clean artifacts

  • Dedicated cleanup runner for artifact directories.
  • Safer target validation and predictable cleanup scope.
  • Completes the full artifact lifecycle directly in IDE.
[Screenshot placeholder: Clean artifacts runner]

Scaffolding and File Templates

  • New Project Wizard with vendor/project validation and subtype selection (validator / lib).
  • Post-init defaults for validator projects:
    • artifacts/ is added to .gitignore,
    • starter test module is created in validators/tests/test_module.ak.
  • Improved project recognition stability after creation and reopen.
  • New → Aiken File with 3 subtypes:
    • Aiken Validator (template resolved from cached/generated placeholder.ak),
    • Aiken Test,
    • Common File (empty .ak).
[Screenshot placeholder: Aiken New Project Wizard]
[Screenshot placeholder: New Aiken File dialog + subtype picker]

Navigation and Editing UX

  • Better usage/declaration popup readability and contextual rendering.
  • Alias imports in use ... as ... now index and resolve consistently.
  • Autocomplete noise reduced by filtering out non-callable test signatures.
  • General UI consistency updates across runner cards, labels, and descriptions.
