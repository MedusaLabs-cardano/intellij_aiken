What's New in Aiken Plugin

Version {{VERSION}}

This update expands the plugin into a much more complete Aiken workflow inside JetBrains IDEs: project scaffolding, file creation, indexing, navigation, usages and rename, parameter info, integrated runners, parameterization, artifact generation, and cleanup.

Workflow Upgrade
The plugin is no longer just editor support plus formatting. It now provides a real end-to-end Aiken workflow inside the IDE.
New Runner Stack
Dedicated Aiken run configurations: Run checks, Build blueprint, Parametrize, Make artifacts, Clean artifacts.
Code Intelligence Overhaul
New module/export/symbol indexing powers smarter completion, navigation, usages, rename, and structure tools.

Runner Integrations

Run checks

  • Three output modes: TTY, JSON, and IDE integrated.
  • IDE mode now builds a real test tree grouped by module and test instead of flattening everything into console text.
  • Warnings and errors are grouped as dedicated sections and keep full Aiken diagnostic context.
  • Per-test details are richer, including traces and execution metrics such as mem/cpu.
  • Selected test rerun is supported where Aiken CLI semantics allow it.
  • Test matching UX was improved, including pattern handling and exact-match behavior.
  • Skip tests now works cleanly in integrated scenarios instead of forcing a degraded flow.
[Screenshot placeholder: Run checks with module tree, tests, warnings, and errors]
[Screenshot placeholder: Per-test details with traces and mem/cpu]

Build blueprint

  • Supports TTY and IDE integrated.
  • Integrated mode behaves like a build runner, not a fake test run.
  • Warnings and errors are preserved as build diagnostics without inventing test nodes.
  • Build watch behavior was aligned with the integrated path.
[Screenshot placeholder: Build blueprint integrated output]

Parametrize

  • Supports both TTY and IDE integrated modes.
  • TTY mode now handles repeated application correctly, including default CBOR queues, in/out tracking, and build invalidation.
  • Auto-repeat can keep applying parameters until no undefined parameters remain.
  • Applied CBOR parameters are collected and shown at the end of the flow for reuse.
  • The new IDE-integrated editor understands nested Aiken schemas: constructors, records, tuples, pairs, lists, maps, booleans, options, and opaque Data.
  • Default CBOR parameters can now be applied behind the scenes first, so the GUI only shows unresolved parameters.
  • Live validation, debounced error highlighting, and better retry behavior make the GUI path practical for real work.
[Screenshot placeholder: Parametrize GUI with nested constructors and collections]
[Screenshot placeholder: Parametrize TTY mode with applied CBOR summary]

Make artifacts

  • A unified pipeline now covers script conversion, mainnet/testnet address generation, policy ID generation, and optional script CBOR display.
  • Each artifact type can be enabled or disabled independently.
  • Artifact filenames are now template-driven, with support for %module%, %validator%, and even subdirectories.
  • Multiple module/validator targets can be processed in a single run.
  • The result view is IDE-integrated, copy-friendly, and summarizes saved files instead of dumping raw CLI output.
[Screenshot placeholder: Make artifacts result view with CBOR, addresses, and policy]
[Screenshot placeholder: Artifact filename templates and saved-file summary]

Clean artifacts

  • Dedicated cleanup runner for artifact directories.
  • Safer target validation and predictable cleanup scope.
  • Completes the full build → parametrize → generate → clean lifecycle inside the IDE.
[Screenshot placeholder: Clean artifacts runner]

Scaffolding and File Templates

New Project Wizard

  • A dedicated Aiken project type was added to the New Project flow.
  • It supports vendor input, project type selection (validator / lib), and token normalization/validation.
  • Project creation now runs aiken new in a way that keeps JetBrains project recognition and Git mapping stable after reopen.
  • Validator projects receive practical post-init defaults such as artifacts/ in .gitignore and a starter test module.
[Screenshot placeholder: Aiken New Project Wizard]
[Screenshot placeholder: Fresh Aiken project with post-init defaults]

New → Aiken File

  • A new file-creation action was added directly to the IDE New menu.
  • Supported subtypes: Aiken Validator, Aiken Test, and Common File.
  • Validator file generation uses a cached/generated template resolved from real Aiken scaffolding instead of a stale hand-written snippet.
  • File names are validated against Aiken-friendly naming rules up front.

Indexing, Navigation, and Editing

Indexation overhaul

  • The plugin now indexes modules, exports, top-level symbols, and function signatures more explicitly.
  • The new model is much better suited to cross-module resolution and project-scale navigation.
  • As a result, completion, usages, rename, declaration resolution, and symbol navigation now operate on a stronger foundation.
[Screenshot placeholder: Module/export/top-level symbol indexing in action]

Navigation and references

  • Go to Declaration became more stable for aliases, locals, shadowed names, and ambiguous targets.
  • Find Usages and Rename now work more consistently across files and modules.
  • The usage/declaration chooser was redesigned to show more useful contextual information.
  • Error and warning nodes produced by integrated runners can now navigate back to source locations.
[Screenshot placeholder: Usage popup with contextual entries]
[Screenshot placeholder: Warning/error navigation back to source]

Completion and imports

  • use-completion now understands module paths, nested modules, and public exports.
  • Completion can insert into use module.{...} blocks correctly and behaves better after . and commas.
  • Alias imports in use ... as ... now index and resolve consistently.
  • Noise was reduced by filtering out non-callable test signatures from normal callable completion contexts.

Structure and parameter info

  • Go to Symbol was added for top-level Aiken declarations.
  • Structure View was added for quick file-level browsing.
  • Ctrl+P / parameter info was expanded to handle more real Aiken call forms.
  • Breadcrumbs and folding support remain available and are now backed by stronger test coverage.

Formatting and polish

  • Formatting now normalizes whitespace-only lines before calling aiken fmt, which fixes real empty-line edge cases.
  • Runner cards, labels, descriptions, and integrated views were polished for denser and more predictable UX.
  • A versioned in-IDE What's New tab was added and is now theme-aware through IDE-provided variables.

This release adds a dedicated runner stack, a project wizard, a new file creation workflow, a much stronger indexing/navigation/search layer, and a practical schema-driven parameterization GUI. Aiken development inside JetBrains IDEs is now faster, deeper, and far more self-contained.
